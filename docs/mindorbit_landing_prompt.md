# 마인드오빗 랜딩페이지 수정 요청

## 작업 개요
기존 랜딩페이지의 **섹션 구성과 카피를 전면 교체**해줘.
디자인 시스템(색상, 폰트, 간격)은 그대로 유지하고, 콘텐츠와 섹션 흐름만 아래 스펙대로 바꿔줘.

---

## 브랜드 & 톤 기준 (수정하지 말 것)
- 브랜드 컬러: `#7F77DD` (purple)
- 핵심 한 문장: **"가볍고 빠르고 아름다운 생각 구조화"**
- 톤: 정돈되고 차분함. 과장 없이 분명하게. 광고처럼 느껴지면 안 됨.
- 피해야 할 표현: 혁신, 강력한 기능, 치유, 협업툴 느낌의 메시지

---

## 섹션 구성 (순서대로 구현)

### 01. Hero Section

**목적:** 방문자가 5초 안에 "이 서비스가 어떤 문제를 해결하는지" 이해하게 만들기

**헤드라인:**
```
생각은 많은데,
정리할 도구가 없었다
```

**서브카피:**
```
메모 앱으로는 부족하고, 협업 툴은 너무 무거운.
Mind Orbit은 그 사이를 채웁니다.
```

**CTA 버튼:**
- Primary: `베타 버전 써보기 →` → 앱 진입 URL 연결
- Secondary: `둘러보기` → Problem 섹션으로 스크롤

**비주얼 요소:**
- 배경에 노드가 천천히 연결되는 CSS 애니메이션 (기존 orbit 애니메이션 유지 또는 개선)
- 가능하면 모바일 앱 목업 이미지 placeholder 추가 (img 태그, src는 비워둬도 됨)

---

### 02. Problem Section

**목적:** "맞아, 나 이래서 불편했어" 공감 유발

**섹션 제목:** `익숙한 도구들, 하지만 이런 적 없었나요?`

**항목 4개 (카드 또는 리스트 형태):**

1. 메모앱에 적었는데, 나중에 찾을 수가 없다
   - 설명: 선형 구조로는 생각의 연결이 보이지 않습니다
2. Miro를 켰다가, 기능에 치여 생각을 잃었다
   - 설명: 생각정리보다 도구 사용이 먼저가 되는 역설
3. 아이디어는 모바일에서 떠오르는데, 도구는 PC에 있다
   - 설명: 맥락이 끊기면 생각도 사라집니다
4. 정리했는데 결과물이 보기 싫어서 다시 안 열었다
   - 설명: 투박한 결과물은 활용되지 않습니다

---

### 03. Positioning Section (신규 추가)

**목적:** "이 서비스가 어디에 있는지" 설명 없이 시각으로 전달

**섹션 제목:** `생각정리 도구의 빈 자리`

**서브카피:** `메모 앱과 전문 협업 툴 사이. Mind Orbit은 그 자리를 채웁니다.`

**비주얼: 가로 스펙트럼 (horizontal spectrum bar)**

아래 구조로 구현해줘:

```
[가벼움] ←————————————————→ [무거움]
[평면적] ←————————————————→ [구조적]

  메모앱        Mind Orbit       Notion   Miro/XMind
  Keep                                    Obsidian
```

구현 방식:
- 가로 막대(bar) 위에 각 도구를 점(dot) + 레이블로 표시
- Mind Orbit 점은 브랜드 컬러(`#7F77DD`)로 강조, 나머지는 회색
- Mind Orbit 점에 pulse 애니메이션 추가
- x축: 왼쪽=가볍고 평면적, 오른쪽=무겁고 구조적
- 막대 아래 구간 설명: 좌측="저관여 도구", 중간="Mind Orbit 타깃", 우측="고관여 도구"

---

### 04. Solution Section (기존 How it works 교체)

**목적:** 기존 도구의 한계를 Mind Orbit이 어떻게 해결하는지 "3가지 약속" 형태로

**섹션 제목:** `Mind Orbit의 세 가지 약속`

**카드 3개:**

**약속 01 — 어디서든 바로 열립니다**
- 본문: 모바일, 태블릿, 데스크탑. 생각이 떠오른 그 자리에서 바로.

**약속 02 — 보기 좋게 정리됩니다**
- 본문: 군더더기 없는 UI. 완성한 맵은 다시 열고 싶어집니다.

**약속 03 — 링크 하나로 공유됩니다**
- 본문: 완성한 마인드맵을 웹 링크로 즉시. 설치 없이 누구나 볼 수 있습니다.

각 카드 아래에 `[앱 화면 스크린샷 placeholder]` img 태그 추가 (src 비워둬도 됨, alt 텍스트만 넣어줘)

---

### 05. Social Proof Section (신규 추가)

**목적:** 신뢰 형성. 단, 과장 없이 솔직하게.

**구현:**
- 숫자 강조 레이아웃:
  - `100명` / 베타 신청, 48시간 만에
  - 설명 문구: "서비스 한 줄도 공개하기 전. 문제에 공감한 사람들이 먼저 손을 들었습니다."
- 이 섹션은 과하지 않게, 심플한 수평 레이아웃으로

---

### 06. Target Section

**목적:** "나를 위한 서비스구나" 인식

**섹션 제목:** `이런 상황이라면, Mind Orbit이 맞습니다`

**항목 4개 (카드 그리드):**

1. 메모앱으로는 부족하고, Notion 열기엔 너무 작은 생각이 있는 분
2. 아이디어가 모바일에서 자주 떠오르는 분
3. 결과물이 보기 좋아야 다시 열게 되는 분
4. 내 생각을 링크 하나로 바로 나누고 싶은 분

---

### 07. CTA Section

**섹션 제목:** `생각이 흩어지기 전에`

**서브카피:** `지금은 베타 단계입니다. 먼저 써보고, 함께 만들어가요.`

**버튼:**
- Primary: `베타 버전 써보기 →`
- Secondary: `둘러보기`

---

### 08. Footer

**포함 내용:**
- 로고 + 서비스명
- 한 줄 설명: "가볍고 빠르고 아름다운 생각 구조화"
- 링크: 소개, 기능, 시작하기, 로그인
- 저작권: `© 2026 Mind Orbit. All rights reserved.`

## 애니메이션 & 인터랙션 요구사항

1. **스크롤 페이드인**: 모든 섹션 진입 시 아래에서 위로 fade-in (기존 로직 유지)
2. **Hero orbit 애니메이션**: 기존 유지 또는 개선
3. **Positioning 스펙트럼**: Mind Orbit 점에 pulse 애니메이션
4. **타이핑 효과**: Hero 또는 Problem 섹션에 유지 (기존 로직 유지)
5. **Sticky nav**: 기존 유지

## 주의사항

- 기존 CSS 변수, 색상 시스템, 폰트 설정은 건드리지 말 것
- 반응형(모바일) 고려: 특히 Positioning 스펙트럼은 모바일에서도 읽혀야 함
- 섹션 번호 순서대로 HTML 구조 유지
- 각 섹션에 id 부여: `#hero`, `#problem`, `#positioning`, `#solution`, `#proof`, `#target`, `#cta`
- 이미지 placeholder는 `<img src="" alt="[설명]" class="mockup-img">` 형태로

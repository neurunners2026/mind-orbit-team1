# 랜딩 네비게이션 통일 및 로그인 상태 전환 계획

## 목표

랜딩 페이지의 네비게이션 바를 랜딩 섹션의 문맥과 통일하고, 로그인 상태에 따라 오른쪽 액션을 자연스럽게 전환한다.

## 확정 범위

- 변경 범위는 `src/components/landing/Navbar.tsx` 중심이다.
- 랜딩 페이지의 섹션 네비게이션만 조정한다.
- 대시보드와 에디터에서 사용하는 `src/components/common/Header.tsx`는 변경하지 않는다.
- 로그인/회원가입 페이지에서 사용하는 닫기(X) 버튼 동작은 유지한다.

## 네비게이션 문구

- Hero 섹션 링크는 제거한다.
- 브랜드 로고 `Mind Orbit`은 기존처럼 `/` 이동 역할을 유지한다.
- 섹션 링크는 영문 Eyebrow 흐름에 맞추되, 상단 네비게이션은 핵심 섹션만 노출한다.
  - `#problem` → `Problem`
  - `#features` → `Features`
- `Target`, `Start`는 랜딩 본문에는 유지하지만 네비게이션에는 표시하지 않는다.

## 인증 상태별 액션

- 인증 로딩 중에는 버튼 영역의 흔들림을 줄이기 위해 placeholder를 렌더링한다.
- 비로그인 상태에서는 `Login` 버튼을 보여주고, 클릭 시 기존 로그인 모달을 연다.
- 로그인 상태에서는 `Dashboard` 링크와 `Logout` 버튼을 나란히 보여준다.
- `Logout` 클릭 시 `signOut()`을 실행하고, 실패하면 콘솔에 에러를 기록한다.

## 테스트 시나리오

- 비로그인 랜딩에서 nav 항목이 `Problem`, `Features`로만 보이는지 확인한다.
- 각 nav 항목 클릭 시 해당 섹션으로 스크롤되는지 확인한다.
- `Login` 클릭 시 기존 로그인 모달이 열리는지 확인한다.
- 로그인 후 오른쪽 액션이 `Dashboard`, `Logout`으로 전환되는지 확인한다.
- `Dashboard` 클릭 시 `/dashboard`로 이동하는지 확인한다.
- `Logout` 클릭 후 다시 `Login` 상태로 전환되는지 확인한다.
- `/login`, `/signup`에서는 기존 X 닫기 버튼이 유지되는지 확인한다.
- `npm run build`로 빌드 성공 여부를 확인한다.
